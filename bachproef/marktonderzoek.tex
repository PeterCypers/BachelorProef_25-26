\section{HuggingFace}
\label{sec:huggingface}
%een github like platvorm waar allerlei LLM-gedreven projecten beschikbaar zijn, dit platvorm is ook een AI-provider met zijn eigen API-key om
%gemakkelijk verschillende modellen te gebruiken en zij spelen dan voor tussenpersoon om de kost van gebruik te regelen (beter verwoorden)

Een github-achtig platvorm waar allerlei AI-gedreven projecten beschikbaar zijn. \\

Uit een interview met Axios\footnote{\href{https://www.youtube.com/watch?v=EmZCC3lIRRU}{https://www.youtube.com/watch?v=EmZCC3lIRRU}} vermeld de CEO van HuggingFace dat ze dubbel het aantal AI repositories hebben als github, met meer dan 6 miljoen op HuggingFace(of H.F.), volgens de host van het interview wordt HuggingFace vaak omschreven als de GitHub van AI \autocite{Rubin2025}. \\

H.F. is een online platvorm waar veel machine learning modellen op te vinden zijn, LLMs, embedding models, LLM training-datasets, repositories zoals op github, maar met iets minder sterke features voor het onderhouden van repositories in het algemeen in vergelijking met github. \\

Het is tegelijk een host van verschillende LLM en ML-modellen en een AI-service provider. Men heeft bijvoorbeeld een huggingface API-token nodig om toegang te krijgen tot de gehoste embedding models. Ze kunnen ook als tussenpersoon gebruikt worden om de kosten van AI-gebruik bij te houden, met inference providers gebruikmakend van HF-token.

    "Hugging Face Inference Providers unifies 15+ inference partners under a single, OpenAI‑compatible endpoint.

    (https://huggingface.co/inference/get-started)
\end{quote}

Postman is een API-testing software waarmee men kan verbinden met APIs, gestructureerde requests sturen en de responses onderzoeken. \\

Postman heeft ook sterke integratie met Visual Studio Code via de gelijknamige extensie, deze bevat gelijkaardige features als men in de online en desktop applicaties vind, API collecties, environments, scripts(waaronder ook testen), de import feature().

MCP servers voorzien een verbinding met externe tools dat AI-agents kunnen gebruiken, model context protocol. Er bestaan veel MCP servers die agents kunnen verbinden met externe tools, Postman heeft een Visual Studio Code plugin dat 'Postman MCP' noemt hiermee kunnen AI-agents verbonden worden met Postman en zo bepaalde acties uitvoeren binnen die software. \\ %todo: check what else MCP can do

\subsection{Synchronisatie van Collections met Postman MCP}
\label{ssec:synchronisatievancollectionsmetpostmanmcp}
Wanneer in projecten reeds API documentatie bestaat in de vorm van een openapi.yaml of openapi.json file, zie sectie: \ref{sssec:oad}, kunnen developers met behulp van Postman MCP verbinding maken met een Postman workspace en de bestaande collecties synchronizeren met de documentatie file(s) welke een single point of truth zijn voor de API van dat project. \\

Door een AI-agent in de IDE aan te spreken en de Postman workspace ID mee te geven in de prompt kan de agent via Postman MCP-tool collections op die workspace bekijken en aanpassen. Zo kan gevraagd worden om de bestaande \textbf{collections} te \textbf{synchroniseren}. Uiteindelijk heeft men zo gebruikmakend van de documentatie en AI-ondersteuning via MCP servers Postman klaar gezet zodat het team verder API-endpoint tests kan doen in Postman. \\

Initieel kan postman een collection aanmaken en opvullen met de \textbf{import feature} en deze kan ook een OpenAPI file (zie 'OAD' sectie: \ref{sssec:oad}) gebruiken om deze import te doen, zie: \ref{fig:postmanimport}

\section{(Volledig) Autonome AI Agents}
Er is steeds meer autonomie mogelijk bij AI-agents, het kan op een schaal gezet worden van hoge menselijke tussenkomst tot helemaal geen menselijk tussenkomst \autocite{mitchell2025}. Wanneer men een vraag stelt aan een online AI-chatbot en met het gegenereerd antwoord zelf tot een besluit komt is er nog volledige controle bewaard over het systeem waarmee de mens interageert. Wanneer de AI-agent inzicht mag hebben in verschillende niveaus van context binnen een systeem en de gebruiker vraagt of er code-aanpassingen mogen gebeuren heeft de AI-agent al veel meer autonomie. Wanneer er geen goedkeuring gevraagd wordt en de AI-agent code kan aanpassen, commits kan maken en pull requests afwerken is er al heel veel autonomie aan de agent gegeven en bestaat het risico dat de menselijk controle over het systeem verdwijnt. Wanneer men toegang verleent tot de AI-agent om zelf volledige projecten af te werken, mogelijks met verhoogde gebruiksrechten over het platform waarin deze werkt kan de mens uit het plaatje verdwenen zijn en heeft de AI-agent alle controle over het systeem. \\

Volgens \textcite{mitchell2025} is het een \textbf{Human} $\leftrightarrow$ \textbf{System} schaal. De mens heeft nog volledige controle wanneer de AI-agent vragen beantwoord, minder controle wanneer de agent bepaalt hoe en wanneer functies worden uitgevoerd, of tools worden gebruikt. Uiteindelijk op het moment dat de AI-agent zelf code mag schrijven en code mag uitvoeren is controle over het systeem volledig uit handen van de mens. Dit soort AI-agents brengen serieuze risico's met zich mee en kunnen best vermeden worden \autocite{mitchell2025}. Zie ook sectie: 'PoLP' (\ref{sec:principleofleastprivilege}). \\

\section{Pricing}

Veel AI-chatbots die men via de browser kan aanspreken hebben een gratis gebruik tot een bepaald gebruikslimiet, bijvoorbeeld na een aantal berichten wordt het model waarmee men spreekt gedegradeerd naar een zwakkere versie maar de vraag- en antwoord service blijft toegankelijk. Het draaien van LLMs is een service die zeker wel met een prijskaartje komt. Als gebruiker die het absoluut minimum nodig wil uitgeven moet je er bewust van zijn dat er een aantal varianten zijn van AI-services. \\

\textbf{Volledig gratis gebruik:} Dit zien we bijvoorbeeld met Github Copilot Chat. Er is een maandelijkse toelage van code-completions en een aparte maandelijkse toelage van chats of agent interacties(dus met tool-gebruik erbij). Bij het aanspreken van de cloud-LLMs die Ollama ter beschikking stelt zien we een "Session use" en een "Weekly usage", het eerste reset op een tijdspanne van uren, het laatste is elke week gereset. Uit ondervinden heeft de Ollama service een veel hoger limiet van gratis gebruik tegenover Github Copilot Chat. Het is mogelijk om heel veel queries te doen op de cloud-LLMs van Ollama zonder een serieuze toename te ondervinden in gebruiks-limiet, na uitbundig gebruik is dit nog nooit de 5\% overschreden. Terwijl bij Github Copilot Chat al snel  over de 10\% zit en er op een maandelijkse reset moet gewacht worden. Github Copilot Chat voorziet de gratis gebruiker van 2000 code-completions en 50 chatsessies per maand\footnote{\href{https://docs.github.com/en/copilot/get-started/plans}{https://docs.github.com/en/copilot/get-started/plans}}.
Dit maakt het wel aantrekkelijker om als volledig gratis gebruiker met de sterkere cloud modellen bij Ollama te werken dan met Copilot Chat. Bij Ollama kan het gebruikslimiet gecontroleerd worden in de settings\footnote{\href{https://ollama.com/settings}{https://ollama.com/settings}}. \\

\textbf{Pay-as-you-go model:}

Sommige services hebben direct een betalingsplan of aankoop van krediet nodig om te beginnen gebruiken. Vooral premium services zoals gebruik van de recentste modellen, zwaardere operaties zoals research, meer geheugen, snellere antwoorden. Men kan bij bepaalde providers zoals Anthropic's Claude eenmalige betalingen maken van een bepaald bedrag en genieten van premium services tot de portefeuille is opgebruikt. API gebruik heeft ook een kost gemeten in dollar kost per miljoen tokens opgesplitst per soort, er is een verschil tussen de kost van input- en output-tokens. \\

\textbf{Subscriptie model:}

Een veiligere optie, gelimiteerd op maandelijks gebruik en betalend per maand met een bepaald verwachte service en specifieke premium gebruiksvoordelen. \\

Als kleine team of startup  kan er zeker een subscriptie genomen worden op een van de verschillende AI services, de individu niveau, de laagste tier van betaling is rond een competitief prijskaartje van \$10/\$19(Github Copilot individueel/enterprise). Het eerste betalend niveau van andere providers kosten al direct wat meer met plannen van rond \$20. In dit opzicht is copilot wel de meest betaalbare keuze. \\

Voor grotere bedrijven met een IT-afdeling kan men beter opteren voor een business/enterprise subscriptie, ook wat security betreft is dit vaak het minimum niveau van abonnement dat men nodig heeft om ervoor te zorgen dat de code niet gebruikt wordt door de AI-providers als trainingsdata. \\
