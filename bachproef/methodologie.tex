\subsection{Keuze van AI-technologie}

Er is geen verder onderzoek gebeurd rond de verschillende mogelijkheden van testen van APIs eerder een focus op bestaande AI technologieën en hoe deze mogelijk kunnen ingezet worden om een zinvolle verbetering in productiviteit teweeg te brengen. Bij het onderzoeken van RAG(Retreival Augmented Generation) werd het snel duidelijk dat efficiënt omgaan met documenten en context voor een AI agent iets van sterk algemene waarde zou kunnen bieden voor ontwikkelaars. Mogelijks ook voor het ontwikkelen en testen van APIs. Daarom is er gekozen om hier een focus op te zetten bij het verdere verloop van dit onderzoek. \\

Uit het onderzoek van \autocite{Golmohammadi2023} werd geconstateerd dat openbaar beschikbare APIs bij vele grote bedrijven niet conform waren met de Open Api Specification (OAS)\footnote{\href{https://swagger.io/specification/}{https://swagger.io/specification/}}. Dit is een wereldwijd erkend conventie voor het ontwikkelen van APIs. Dit is ook een online beschikbaar document dat richtlijnen en templates omschrijft over hoe men best te werk bij het ontwerpen en ontwikkelen van APIs. \\

\subsection{Het nut van RAG bij API design}

Retrieval Augmented Generation, een machine learning techniek dat AI-agents kan ondersteunen bij het efficiënt omgaan met documenten en developers zou kunnen ondersteunen bij de interpretatie van delen van de codebase, zoals API-templates, API test-sjablonen, misschien ook een lokaal en bedrijf-specifieke OAD (Open Api Document). Een AI agent die dit kan interpreteren en hierbij vragen kan gesteld worden zou zeker een zinvol uitwerking hebben, API design en testing waar structuur en best-practice vaan groot belang zijn, een AI-Agent die hierover up-to-date en context-onderbouwde antwoorden kan bieden, met behulp van RAG-technieken, zou mogelijks een goede ondersteuning zijn voor developers en bijdragen aan een verhoogde werk-efficiëntie. \\

Er wordt een onderscheid gemaakt tussen het onderzoeken en begrijpen van de OAS, waar we niet verder op ingaan en het zoeken naar een AI-technologie die deze kan helpen interpreteren, dit wordt wel verder grondig onderzocht. \\

\subsubsection{Alternatieven}
Andere onderzoeksrichtingen die eveneens tot nuttige AI-toepassingen kunnen leiden, zijn onder meer:

\begin{itemize}
    \item \textbf{MCP:} Onderzoeken welke MCP servers mogelijk leiden tot nuttige tools, bijvoorbeeld: \textit{Postman MCP}
    \item \textbf{Workflow Automation Platvormen:} Bijvoorbeeld \textbf{n8n} Een no-code/low-code dat men toelaat AI-agent geïntegreerde workflows te creëren gebruikmakend van nodes die stappen of acties in het proces voorstellen en met elkaar verbonden kunnen worden.
\end{itemize}

Er werd gekozen voor een meer low-level aanpak met RAG om tot een diepere, algemene inzicht te komen in het verhaal van AI-agents en LLMs.

\label{sec:agentinzettenentesten}
